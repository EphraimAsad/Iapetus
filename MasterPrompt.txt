======================================
PRODUCT PURPOSE
======================================

Iapetus is a Shelf-Life Risk Engine.

The user enters food microbiology challenge-test style conditions and gets back:
1. predicted microbial growth curve over time
2. Monte Carlo uncertainty band
3. threshold exceedance probability
4. clear decision and recommendation
5. short AI-generated plain-English summary

This is an exploratory simulation-based MVP, not a regulatory-grade validated system.
Architecture must allow future replacement of synthetic data with real challenge-test data using the same schema.

Primary organism:
- Listeria monocytogenes

Primary threshold:
- 100 CFU/g

======================================
PHASE 4 — GROWTH CURVE GENERATION SERVICE
======================================

Implement:
backend/app/services/curve_service.py

This service must:
- accept a scenario input
- construct a day grid, default like:
  [0, 1, 3, 5, 7, 10, 14, 21, 28, 35, 42]
  but adapt to target_shelf_life_days if needed
- create feature rows for each day
- call the regression model
- return predicted log CFU/g values by day

Return structure:
{
  "days": [...],
  "predicted_log_cfu_g": [...]
}

If sensible, apply light smoothing or constraints, but keep it transparent.
Avoid wild erratic jumps if possible.

======================================
PHASE 5 — MONTE CARLO UNCERTAINTY ENGINE
======================================

Implement:
backend/app/services/monte_carlo_service.py

This service must:
- perturb uncertain scenario inputs within realistic bounds
- generate many simulated curves
- summarize uncertainty

Default perturbation assumptions:
- ph ± 0.08
- aw ± 0.01
- storage_temperature_c ± 0.7
- initial_inoculum_log_cfu_g ± 0.25
- salt_percent ± 0.15
- sugar_percent ± 0.2
- fat_percent ± 0.5

Use bounded sampling so values stay realistic:
- aw must remain in valid food range
- ph should remain sensible
- percentages should not go negative

MVP default number of simulations:
- 500

Outputs:
- days
- median_log_cfu_g
- p10_log_cfu_g
- p90_log_cfu_g
- threshold_exceedance_probability_by_day

Threshold is 100 CFU/g.
Convert appropriately if working in log space.

Also return high-level uncertainty driver indicators if practical.

======================================
PHASE 6 — DECISION ENGINE
======================================

Implement:
backend/app/services/decision_service.py

This service must consume:
- target shelf life
- Monte Carlo outputs
- classifier output if available

It must return:
- growth_risk_class
- threshold_exceedance_probability
- recommended_max_shelf_life_days
- challenge_test_recommended
- confidence_label
- threshold_cfu_g

Initial suggested logic:
- Low risk: exceedance probability < 5%
- Moderate risk: 5% to <20%
- High risk: 20%+

- challenge_test_recommended = true if probability at target shelf life >= 10%
- challenge_test_recommended = true if uncertainty is wide and borderline
- recommended_max_shelf_life_days = latest day where exceedance probability remains below a conservative threshold, default 5%

Always indicate that outputs are simulation-based for MVP.
